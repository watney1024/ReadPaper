# 批量阅读并总结文献

当用户输入 `/批量阅读并总结文献` 时，严格按以下步骤执行。

## 项目根目录

Claude Code 启动时已自动切换至项目根目录（即包含 `CLAUDE.md` 的目录）。
后续所有相对路径均以此为基础。

## 参数解析

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--from 路径` | MD 来源文件夹 | `md_unread` |
| `--md-to 路径` | 处理完成后 MD 的归档文件夹 | `md_done` |
| `--docx-to 路径` | MD 笔记输出文件夹（沿用历史参数名，实际输出 `.md`） | `docx_done` |

相对路径均相对于项目根目录。

## 执行步骤

### 第 1 步：确定路径

- 来源文件夹 C = `--from` 值 或 当前目录下的 `md_unread`
- MD 归档文件夹 D = `--md-to` 值 或 当前目录下的 `md_done`
- MD 笔记输出文件夹 E = `--docx-to` 值 或 当前目录下的 `docx_done`

### 第 2 步：列出所有待处理 MD 文件

使用 Glob 工具列出文件夹 C 中所有 `.md` 文件，按字母顺序排列。

若没有 MD 文件：告知用户 "【文件夹C】中没有待处理的 MD 文件"，停止执行。

若有文件：告知用户 "找到 X 篇待处理文献，开始逐篇处理..."

### 第 3 步：逐篇处理

对每一篇 MD 文件，完整执行 `/阅读并总结文献` 的 **第 3 步到第 8 步**（提取标题、分析、生成 MD 笔记、归档），使用相同的文件夹 C、D、E 路径。

每篇完成后立即输出进度：

```
[X/Y] ✓ 已完成：【论文标题】→ 【MD笔记路径】
```

若某篇处理中出现错误，输出：

```
[X/Y] ✗ 处理失败：【MD文件名】
错误信息：【错误详情】
```

继续处理下一篇，不中断整体流程。

### 第 4 步：汇总报告

所有论文处理完成后，输出：

```
══════════════════════════════
批量处理完成
  成功：X 篇
  失败：Y 篇（若有，列出文件名）
  MD  笔记保存在：【文件夹E路径】
  MD  归档在：【文件夹D路径】
══════════════════════════════
```
